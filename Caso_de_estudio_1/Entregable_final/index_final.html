Club Cinepolis

Resumen Situación Actual

Club Cinépolis evoluciona de transacciones anónimas a una arquitectura basada en datos. Cada compra es ahora un evento identificable que alimenta sistemas de analítica y marketing, transformando al usuario en un activo clave del negocio.

1. Identidad (Okta CIC)

Autenticación centralizada en app, web y POS. Reduce fraude y duplicados.

2. Operativa (POS)

Registro en tiempo real de ventas en taquilla y dulcería. Punto de entrada de datos.

3. Motor de Lealtad

Lógica de puntos (5%/10%), niveles de socio y validación de promociones.

4. Base de Datos Central

Almacén de usuarios, transacciones, canjes y reglas de expiración.

5. Capa Analítica

Segmentación, predicción de comportamiento y campañas personalizadas.

Arquitectura del Sistema

Estrategia Detrás del Sistema

El diseño está optimizado para transformar el comportamiento del usuario mediante gamificación y control financiero.

📈

Frecuencia

Modelo semestral por niveles diseñado para incrementar las visitas constantes al cine.

💰

Gasto Promedio

A mayor acumulación de puntos, mayor es el incentivo de consumo en dulcería y taquilla.

📊

Big Data

Generación masiva de datos de comportamiento para toma de decisiones estratégicas.

🎮

Gamificación

Sistema de niveles (Fan a Súper Fan) que impulsa al usuario a "subir de categoría".

🕒

Pasivos

El vencimiento de puntos reduce pasivos financieros y genera picos de consumo a fin de año.

           from datetime import datetime

            from datetime import datetime

            class Usuario:
                def __init__(self, id_usuario, nombre):
                    self.id = id_usuario
                    self.nombre = nombre
                    self.visitas = 0
                    self.puntos = 0
                    self.nivel = "Fan"
                    self.historial = []

            # Actualizar nivel según visitas
            def actualizar_nivel(usuario):
                if usuario.visitas >= 24:
                    usuario.nivel = "Super Fanatico"
                elif usuario.visitas >= 12:
                    usuario.nivel = "Fanatico"
                else:
                    usuario.nivel = "Fan"

            # Calcular puntos
            def calcular_puntos(usuario, monto):
                if usuario.nivel == "Fan":
                    tasa = 0.05
                else:
                    tasa = 0.10
                return monto * tasa

            # Registrar transacción
            def registrar_compra(usuario, monto):
                fecha = datetime.now().date()

                # Validar una visita por día
                if not any(t["fecha"] == fecha for t in usuario.historial):
                    usuario.visitas += 1

                # Actualizar nivel
                actualizar_nivel(usuario)

                # Calcular puntos
                puntos_ganados = calcular_puntos(usuario, monto)
                usuario.puntos += puntos_ganados

                # Guardar transacción
                usuario.historial.append({
                    "fecha": fecha,
                    "monto": monto,
                    "puntos": puntos_ganados
                })

                print(f"\nUsuario: {usuario.nombre}")
                print(f"Nivel: {usuario.nivel}")
                print(f"Compra: Q{monto}")
                print(f"Puntos ganados: {puntos_ganados}")
                print(f"Total puntos: {usuario.puntos}")

            #Expiración de puntos
            def expirar_puntos(usuario):
                usuario.puntos = 0

            #Canje de recompensas
            def canjear(usuario, costo):
                if usuario.puntos >= costo:
                    usuario.puntos -= costo
                    print("Canje exitoso")
                else:
                    print("Puntos insuficientes")

                if dia == "martes":

Seguridad

Sistema de Seguridad Actual

Identidad Centralizada: Uso de Okta para separar la operación del cine de la gestión crítica de usuarios.

Comunicaciones Cifradas: Túnel seguro entre puntos de venta (Vista Cinema) y la nube.

Prevención de Fraude: Motor de reglas estáticas para evitar duplicación de puntos y visitas.

Consejo de Mejora: Análisis Dinámico

Implementar un sistema de Machine Learning que aprenda los hábitos del socio para detectar anomalías (geolocalización o picos de gasto) en tiempo real.

Conclusiones

1. El dato es el activo principal

El valor real no reside en los puntos acumulados, sino en la información estratégica generada:

  • Hábitos de consumo.
  • Frecuencia de visita.
  • Preferencias de usuario.

2. Arquitectura orientada a eventos

Cada compra es un evento crítico que desencadena acciones en tiempo real:

  • Procesamiento inmediato.
  • Sincronización de sistemas.
  • Alimentación de analítica.

3. Gamificación como estrategia

El sistema de niveles es la herramienta clave para el crecimiento del negocio:

  • Comportamiento recurrente.
  • Generación de engagement.
  • Aumento de retención.
